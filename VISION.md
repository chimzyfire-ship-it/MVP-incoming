The AI Agent Pipeline — What Happens When You Click Run
This is the invisible intelligence that makes GitMorph possible. When a user clicks Run on any
tool the following four-stage pipeline executes automatically.
Stage 1 — Static Analysis
The agent reads everything available about the repo: README.md, Dockerfile, docker-
compose.yml, package.json, requirements.txt, .env.example, and any setup documentation. It
builds a complete understanding of what the tool needs before attempting to run anything.
GitMorph Master Document • April 2026 PageGitMorph | The OS for Open Source Software | CONFIDENTIAL
Stage 2 — Configuration Generation
Based on the static analysis the agent infers all required services (databases, caches,
queues), environment variables with sensible defaults, build commands, port mappings, and
resource requirements. It auto-generates a Dockerfile if one does not exist.
Stage 3 — Build with Iterative Error Recovery
The agent attempts to build and run the tool. If it fails it captures the error, feeds it back to the
LLM for diagnosis, and attempts a fix. This iterative loop runs up to 5 times before graceful
failure. The fallback chain: existing Dockerfile → Nixpacks auto-detection → community
buildpack → AI-generated Dockerfile.
Stage 4 — Health Check and Result Delivery
The agent verifies the service responds correctly, scans expected ports if needed, performs a
basic functionality test, and delivers the result to the user. The Active Theory style loading
animation plays throughout. When complete the animation converges and the tool interface
opens.
